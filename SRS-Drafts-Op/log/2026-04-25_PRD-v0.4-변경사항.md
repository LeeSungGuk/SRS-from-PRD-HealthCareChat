- 작업일: 2026-04-25
- 목적: `실버케어초안_PRD_v0.3.md`를 기준으로 B2B2C API 플랫폼 PRD를 SRS 전환 가능한 수준까지 보강

## 1. 사용자 스토리 및 AC 확장

### 변경 내용

기존 v0.3의 사용자 스토리는 B2B 제조사와 독거 어르신 중심으로만 구성되어 있었다. v0.4에서는 B2B2C 구조에 맞게 다음 페르소나별 스토리와 수용 기준을 추가했다.

- B2B-DEV: 연동 개발자
- B2B-PM: 제품/사업 담당자
- A: 독거 어르신
- B: 보호자 자녀
- C: 요양기관 관리자
- D: 플랫폼 운영자

### 핵심 판단

B2B2C 제품에서는 돈을 내는 고객, API를 붙이는 개발자, 실제 사용하는 어르신, 리포트를 보는 보호자/기관의 성공 조건이 다르다. 따라서 사용자 스토리를 단일 사용자 중심으로 두면 SRS에서 권한, 데이터, 리포트, 운영 요구사항이 누락될 가능성이 크다.

## 2. MVP In Scope / Out of Scope 추가

### 변경 내용

`1-4. Phase A MVP 범위`를 추가해 MVP 목적, 검증 질문, In Scope, Out of Scope를 명시했다.

### In Scope 요약

- B2B API 인증
- `/api/v1/chat/reply` 대화 생성
- 제한적 메모리
- 정서/위험 신호 태깅
- 안전 응답 정책
- 일반 안부 중심 선제 발화
- 개발자 문서/샌드박스
- PoC 집계 리포트
- 운영 모니터링
- 과금 검증

### Out of Scope 요약

- 자체 하드웨어 제조
- STT/TTS 음성 처리
- 보호자/기관 완성 앱 직접 제공
- 의료 진단, 처방, 치료 조언
- 응급 구조 보장
- 119/응급기관 자동 연결
- 센서 데이터 수집/분석
- 치매 조기 진단/낙상 판정
- 고도화 관리자 대시보드
- 파트너별 커스텀 LLM 파인튜닝
- 정식 가격표 최적화

### 핵심 판단

MVP는 완성형 돌봄 플랫폼이 아니라, B2B 파트너 기기에 텍스트 기반 시니어 대화 API를 붙였을 때 실제 사용성, 응답 속도, 안전성, 유료 전환 가능성을 검증하는 단계로 정의했다.

## 3. 비즈니스 모델 및 과금 원칙 추가

### 변경 내용

`1-5. 비즈니스 모델 및 과금 원칙`을 추가했다.

### 과금 구조

- PoC: 1~3개월 무료 또는 저가 고정비
- Core 정식 전환: 월 기본료 + 월 활성 디바이스 수 기반 과금
- Usage Overage: 기본 제공 API 호출량 초과분 사용량 과금
- Premium / Phase B: 센서 통합, 보호자/기관 리포트, 고급 위험 분석 추가 과금

### PoC 유료 전환 조건

PoC 종료 후 다음 조건 중 3개 이상 충족하면 유료 전환을 제안하도록 정리했다.

- 인당 일평균 대화 API 호출 5회 이상
- `/api/v1/chat/reply` p95 800ms 이하
- 파트너 개발자의 연동 만족도 4/5 이상
- 정식 탑재 예상 디바이스 수 500대 이상 또는 내부 상품화 검토 착수
- 보호자/기관 리포트 또는 정서/위험 신호 요약에 대한 긍정 피드백
- 의료/응급/자해 발화에서 안전 응답 정책 정상 발동

### 핵심 판단

단순 API 호출당 과금을 전면에 두면 파트너가 사용량 증가를 비용 리스크로 볼 수 있다. 따라서 MVP에서는 활성 디바이스 기반 월 과금을 중심으로 수용성을 검증하고, 초과 사용량 과금은 비용 방어 장치로 둔다.

## 4. 개인정보 동의/보존/삭제 정책 초안 추가

### 변경 내용

`5-3. 개인정보 동의/보존/삭제 정책 초안`을 추가했다.

### 추가한 기준

- 데이터 분류
- 기능별 동의 범위
- 역할별 접근 권한
- 보존/삭제 원칙
- 전파 삭제 범위
- 위탁, 제3자 제공, 재위탁 경계
- 자동화된 판단과 사람의 검토
- 금지 원칙

### 중요한 보정

성인 어르신의 경우 보호자가 자동으로 대리 동의권을 갖는다고 보지 않고, 기본 원칙은 어르신 본인 동의로 정리했다. 법정대리인 또는 위임자가 있는 경우에만 대리 권한을 인정한다.

또한 통증, 수면, 복약, 우울감, 응급 상황 관련 표현은 민감 건강정보에 준하는 높은 보호 수준으로 취급하도록 했다.

### 핵심 판단

이 제품은 단순 챗봇이 아니라 어르신의 건강, 정서, 위험 신호를 다루므로 개인정보 정책이 SRS 데이터 모델과 권한 모델의 기반이 된다. 특히 B2B 파트너, 실버케어, 외부 LLM/클라우드 제공자의 위탁/재위탁 관계를 문서에 명시했다.

## 8. 리스크 및 ADR 추가

### 추가된 리스크

- R6: 외부 LLM/클라우드 위탁 및 재위탁 리스크
- R7: 평가셋 부실로 인한 안전 회귀

### 추가된 ADR

- ADR-005: 대화 원문은 기본적으로 보호자/기관에 제공하지 않고 요약 리포트 중심으로 제공

### 핵심 판단

어르신의 프라이버시 보호와 B2B2C 책임 경계를 위해 원문 기본 제공을 막고, 보호자/기관에는 확인이 필요한 정서·위험 신호 중심으로 제공하는 방향을 고정했다.

## 9. SRS 전환 시 남은 주요 질문

현재 PRD는 SRS 전환 가능한 수준까지 많이 보강되었지만, 다음 항목은 SRS에서 확정해야 한다.

- P0 API의 상세 OpenAPI schema, 필드 타입, 길이 제한, pagination 방식
- 메모리 저장 동의, 보존 기간, 삭제 정책
- 대화 원문, 요약 메모리, 위험 태그, 리포트, 로그의 전파 삭제 범위
- B2B 파트너, 실버케어, 외부 LLM/클라우드 제공자의 처리자/수탁자/재수탁자 관계
- `risk_level`, `emotion_tags`, `recommended_action`의 최종 라벨링 기준과 평가셋 기준
- AI 평가셋의 최소 샘플 수, 카테고리 비율, 라벨링 리뷰어 기준
- 의료/처방/응급 발화에 대한 금칙 응답과 안전 응답 문구
- 월 활성 디바이스와 기본 제공 API 호출량 측정/리포트 방식
- 외부 LLM 제공자에게 전송되는 필드, 보관 여부, 학습 사용 금지, 국외 이전 여부
- PoC 리포트 API/CSV의 필드, 갱신 주기, 다운로드 권한
- Phase B 센서 데이터의 최소 표준 스키마
